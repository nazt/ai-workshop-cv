### Voice System Analysis

Nat ถามว่าทำไม weyermann voice system ดีกว่า - วิเคราะห์เปรียบเทียบ:

**ระบบเรา (Simple)**:
- Hardcoded voice/rate ใน hook
- ไม่มี queueing (เสียงซ้อนกัน)
- แค่บอก "done" ไม่มี context
- ไม่รู้ว่าใครพูด

**ระบบ Weyermann (Production-grade)**:
- TOML config file - เปลี่ยนเสียงง่าย
- Voice per agent - รู้ว่าใครพูด
- Speech queue with atomic `mkdir` lock - ไม่ซ้อนกัน
- อ่าน transcript - บอก context จริงๆ
- Desktop notifications + logging
- Full agent lifecycle (start/complete/lock/status)

**บทเรียน**: ระบบเราเป็น prototype ดีสำหรับเรียนรู้ แต่ถ้าจะ run หลาย agents ต้องใช้แบบ Weyermann
