## Lesson Learned: GitHub Markdown

### ปัญหา
ใช้ 5 commits แค่จะ format header ให้แยกบรรทัด

### สาเหตุ
GitHub markdown **ไม่ render line breaks** แบบ plain text ต้องใช้วิธีอื่น

### วิธีที่ไม่ work บน GitHub

```markdown
# ❌ plain text ติดกัน
**วันที่**: 7 ธันวาคม
**เวลา**: 14:50

# ❌ blockquote
> บรรทัด 1
> บรรทัด 2

# ❌ dot separator
**วันที่** · **เวลา** · **session**
```

### วิธีที่ work บน GitHub

```markdown
# ✅ Bullet list (แนะนำ)
- **วันที่**: 7 ธันวาคม
- **เวลา**: 14:50

# ✅ Table
| Key | Value |
|-----|-------|
| วันที่ | 7 ธันวาคม |

# ✅ HTML <br>
**วันที่**: 7 ธันวาคม<br>
**เวลา**: 14:50

# ✅ 2 spaces ท้ายบรรทัด (ยากเห็น)
**วันที่**: 7 ธันวาคม
**เวลา**: 14:50
```

### Rule of Thumb
> เวลาต้องการหลายบรรทัดใน GitHub markdown → **ใช้ bullet list** เป็นตัวเลือกแรก

### ครั้งหน้าต้องทำ
1. ถาม user ก่อนว่าอยากได้ format แบบไหน
2. รู้ข้อจำกัดของ platform ก่อนเขียน
3. Preview ก่อน commit ถ้าไม่แน่ใจ
